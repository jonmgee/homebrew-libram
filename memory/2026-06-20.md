# 2026-06-20 — Full project state snapshot

## Summary

All 14 entry types implemented in the create form. Sub-category browse system complete. PHB-inspired fantasy styling applied. Bug fixes deployed. Project docs updated to reflect current state.

## Entry types implemented (chronological)

### 2026-06-17 — Initial form (magic_item & weapon)
- `magic_item` — rarity (6-tier dropdown), requires_attunement toggle, item_subtype, charges
- `weapon` — damage_dice, damage_type (slashing/piercing/bludgeoning), bonus (+0 to +3), properties, cost, weight
- Shared fields: name, description, source, dm_only toggle, tags (comma-separated), campaign
- Type-specific fields stored in JSONB `properties` column
- RLS: anon INSERT policy added (temp measure)
- Confirmed working: test entry landed in Supabase dashboard

### 2026-06-17 — Armour, potion, adventuring gear, trinket
- `armour` — armour_type (light/medium/heavy/shield), bonus, stealth_disadvantage, cost, weight
- `potion` — effect, duration, rarity
- `adventuring_gear` — gear_category (tool/kit/container/consumable/focus/instrument/other), quantity, properties, cost, weight
- `trinket` — description only (no type-specific fields)
- Removed `base_ac` from armour form (not needed — AC is derived from armour type)
- Removed `charges` from potion form (not relevant)

### 2026-06-17 — Spell & scroll
- `spell` — level (cantrip–9), school (8 schools), casting_time, range, components (V/S/M checkboxes), material, duration, classes, ritual toggle, concentration toggle
- `scroll` — spell_name, spell_level, rarity

### 2026-06-17 — Monster & NPC
- `monster` — CR, size (tiny–gargantuan), creature_type (14 types), alignment, AC, HP, speed, all 6 ability scores, saving_throws, skills, damage_resistances, damage_immunities, condition_immunities, senses, languages, actions, legendary_actions, special_abilities
- `npc` — all monster fields + role + faction
- Ability scores default to 10

### 2026-06-17 — Delete with confirmation
- Browse page: per-entry Delete link → "Delete? Yes / No" confirmation
- Error feedback on delete failure

### 2026-06-17 — Duplicate detection
- On save: checks for existing entry with same name + type
- Shows "Save anyway" override button
- Uses `allowDuplicate` ref to bypass check on second attempt

### 2026-06-17 — Form refactor
- Fixed form state bleeding on type switch (reset to EMPTY_FORM)
- Fixed validation edge cases
- Refactored duplicate check into shared logic

### 2026-06-18 — Sub-category flow
- `src/lib/subcategories.ts` — maps entry types to human-readable sub-category labels
- `SubCategoryPage` at `/browse/:category` — card grid with real entry counts from Supabase
- `BrowsePage` accepts subcategory param for filtered entry lists
- Create form auto-populates subcategory field (stored in properties.subcategory)
- "Misc" sub-category catches unclaimed types

### 2026-06-18 — Sub-category fixes
- Entry counts now come from Supabase, not hardcoded type counts
- Subcategory field switched to disabled dropdown with human labels ("Wondrous Items" not "wondrous_items")

### 2026-06-19 — Background, Feat, Subclass
- `background` — skill_proficiencies, tool_proficiencies, languages, feature_name, feature_description, equipment, personality_traits, ideals, bonds, flaws
- `feat` — prerequisite, benefit
- `subclass` — parent_class (13 classes), subclass_features

### 2026-06-19 — Table type
- `table` — die dropdown (d4/d6/d8/d10/d12/d20/d100), table_category text, dynamic row builder (add/remove roll_range + result pairs)

  - Google Fonts: Cinzel (headings), IM Fell English (body)

- Route `/browse/all` now routes directly to BrowsePage (commit 2c10999)

## Database

### Migration 001 (2026-06-16)
- `entries` table: id (uuid PK), name, type, description, source, dm_only, tags (text[]), campaign, created_at, properties (jsonb)
- Indexes: GIN on full-text search (name+description), GIN on tags, GIN on properties, B-tree on type, campaign, dm_only, created_at DESC
- RLS: anon = SELECT, authenticated = full CRUD

### Migration 002 (2026-06-19)
- Added DELETE grant + policy for anon key (so browse page can delete without auth)
